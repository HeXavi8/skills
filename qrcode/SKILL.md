# QRCode

Generate QR codes from text or URLs. The CLI supports SVG, PNG, JPG outputs. You can control dark color, background color, module shape (square or circle), finder-eye style (square, circle, rounded), error correction, output size, scale and image quality.

## Generate

```bash
# SVG (write to file)
node {baseDir}/scripts/generate.mjs "Hello world" --format svg -o hello.svg --size 1024 --dot circle --eye rounded --color "#0a0a0a" --background "#ffffff"

# SVG to stdout
node {baseDir}/scripts/generate.mjs "https://example.com" --format svg

# PNG (transparent background)
node {baseDir}/scripts/generate.mjs "https://example.com" --format png -o example.png --size 1024 --scale 2 --dot circle --transparent

# JPG (flatten with background, set quality)
node {baseDir}/scripts/generate.mjs "https://example.com" --format jpg -o example.jpg --size 1024 --quality 85 --background "#ffffff"
```

## Options

- `-o, --out <path>`: output file (if omitted and format=svg, SVG is written to stdout)
- `--format <svg|png|jpg>`: output format (default: svg). If `png` or `jpg` is selected an output path must be provided.
- `--size <px>`: base pixel size for the image before scaling (default: 1024)
- `--scale <n>`: multiplier applied to `--size` to produce higher resolution (default: 1)
- `--margin <modules>` or `--quiet <modules>`: quiet zone in QR modules (default: 4)
- `--dot <square|circle>`: module (dot) shape (default: square)
- `--eye <square|circle|rounded>`: finder-eye shape (default: square)
- `--color <hex|rgb>`: dark color (default: #000000)
- `--background <hex|rgb>`: background color (default: #ffffff). Ignored when `--transparent` is used for PNG.
- `--transparent`: when producing PNG, leave background transparent instead of filling with `--background` (no effect for SVG/JPG)
- `--ec <L|M|Q|H>`: error correction level (default: M)
- `--quality <1-100>`: JPEG quality (default: 80)

Notes on behavior:

- `svg` output can be printed to stdout if `--out` is omitted. For `png`/`jpg` the script requires `--out <file>` because binary output is written to a file.
- `--scale` is useful for producing high-DPI images: e.g. `--size 1024 --scale 2` -> 2048px image.

## Notes

Installation

- This skill requires Node.js. From the skill folder run:

```bash
cd {baseDir}
npm install
```

- The package installs two main dependencies:
  - `qrcode` — to compute the QR matrix
  - `sharp` — to convert SVG to PNG/JPG. `sharp` is a native module that downloads prebuilt binaries; the install may compile on some platforms. On macOS this usually works out-of-the-box, but if you see compilation errors, check `sharp` docs for platform-specific requirements.

Examples / Quick recipes

- Basic SVG to stdout:
  `node {baseDir}/scripts/generate.mjs "https://example.com"`
- High-resolution PNG (transparent):
  `node {baseDir}/scripts/generate.mjs "https://example.com" --format png -o hi-res.png --size 1024 --scale 3 --dot circle --transparent`
- JPG for print (flatten with white background):
  `node {baseDir}/scripts/generate.mjs "https://example.com" --format jpg -o print.jpg --size 2048 --quality 95 --background "#ffffff"`

Troubleshooting

- If `npm install` fails due to `sharp` building errors, try installing system prerequisites (Xcode Command Line Tools on macOS) or consult https://sharp.pixelplumbing.com/install.
- If the QR does not scan, increase `--size` or `--scale`, reduce visual modifications, or increase error correction via `--ec Q` or `--ec H`.

Notes

- Output is SVG by default and fully editable; PNG/JPG are generated by rasterizing that SVG. For more sophisticated styles (gradients, logos), consider editing the SVG output or request additional rendering features.
